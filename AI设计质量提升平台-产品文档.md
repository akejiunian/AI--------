## 1. 文档概述

### 1.1 编制背景

市政设计行业正面临以下核心挑战：

- **设计质量管控难度大**：项目类型多样（道路、桥梁、排水、规划等），涉及大量技术规范与标准图集，人工校审难以覆盖全部质量风险点。
- **重复性工作占比高**：投标文件编制、图纸审查、文本校审、方案设计等环节存在大量重复劳动，严重制约设计效率。
- **知识与经验流失**：核心技术人员经验难以体系化沉淀，人员流动导致设计能力反复"归零"。
- **行业竞争压力加剧**：竞争对手已应用AI技术实现3000页高质量标书自动编制等突破，传统工作模式面临淘汰风险。
- **数据安全约束严格**：涉密工程数据不得上传公有云，对AI能力的私有化部署提出刚性要求。

### 1.2 平台定位

**AI设计质量提升平台**是面向市政类设计院的一体化智能设计质量管控平台。平台以"**知识驱动 + 智能工具**"为双核，构建覆盖设计全流程的AI赋能体系，实现从知识沉淀、质量管控到设计提效的全链条升级。

### 1.3 建设目标

| 目标维度 | 具体指标 |
|---------|---------|
| 质量提升 | 设计文件差错率降低 60% 以上，校审覆盖率达到 100% |
| 效率提升 | 投标文件编制时间缩短 70%，图纸审查效率提升 50% |
| 知识沉淀 | 历史项目可复用率达到 80%，建立全院统一知识底座 |
| 数据安全 | 全部AI能力私有化部署，涉密数据零外泄 |
| 标准统一 | 全院设计质量标准统一管控，各分院协作规范一致 |

### 1.4 目标用户

| 用户角色 | 使用场景 | 核心需求 |
|---------|---------|---------|
| 院长/总工 | 全院质量管控、技术决策 | 全局质量看板、知识资产总览 |
| 分院总工/技术负责人 | 分院质量管理、方案审批 | 分院质量数据看板、技术标准检索 |
| 项目负责人/专业负责人 | 项目投标、方案设计、质量把控 | 标书生成、方案辅助、校审管理 |
| 设计工程师 | 日常设计、制图、文档编制 | CAD辅助、文本校审、规范查询 |
| 校审人员 | 图纸审查、文本校审 | 自动化检查工具、审查报告生成 |
| 知识管理员 | 知识库维护、模板管理 | 知识条目管理、模板更新维护 |

## 3. AI知识库板块

AI知识库是整个平台的知识底座，为AI工具库的各项功能提供数据支撑。三大子库共同构成"**标准—经验—案例**"三位一体的知识体系。

### 3.1 技术库

> **定位**：整合所有技术规范、标准图集及材料数据库，构建标准化知识底座，是设计工作的"权威依据源"。

#### 3.1.1 功能架构

```
技术库
├── 规范标准库
│   ├── 国家标准（GB）
│   ├── 行业标准（CJJ、JGJ等）
│   ├── 地方标准（各省市地方规范）
│   ├── 企业内部标准（院级技术规定）
│   └── 标准状态管理（现行/废止/替代关系）
│
├── 标准图集库
│   ├── 国家建筑标准设计图集
│   ├── 市政专用标准图集
│   ├── 院级通用图集
│   └── 图集版本管理
│
├── 材料数据库
│   ├── 市政材料参数库（沥青、混凝土、管材等）
│   ├── 材料价格信息库（联动造价库）
│   ├── 新材料/新工艺库
│   └── 供应商信息库
│
└── 智能检索引擎
    ├── 语义检索（自然语言问答）
    ├── 关联推荐（规范交叉引用）
    ├── 版本追踪（自动提示更新/废止）
    └── 全文检索（关键词精确匹配）
```

#### 3.1.2 核心功能清单

| 功能模块 | 功能点 | 功能说明 | 优先级 |
|---------|-------|---------|-------|
| **规范管理** | 规范入库与解析 | 支持PDF/Word格式规范自动解析入库，提取目录结构、条文内容、适用范围 | P0 |
| | 版本管理 | 记录规范的历次修订版本，维护"替代/废止"关系链，自动标注当前有效版本 | P0 |
| | 规范更新提醒 | 新版规范发布后自动推送通知至相关专业负责人，标注变更内容 | P1 |
| **图集管理** | 图集数字化 | 将纸质/扫描版图集数字化，支持按图号、专业、部位等维度检索 | P1 |
| | 图集引用追踪 | 统计各图集在项目中的引用频次，辅助评估图集使用价值 | P2 |
| **材料数据库** | 材料参数查询 | 支持按材料类别、性能指标、适用场景等多维度查询 | P0 |
| | 价格信息维护 | 定期更新材料市场指导价，支撑造价估算与方案比选 | P1 |
| | 新材料推荐 | 基于项目类型和地域自动推荐适用的新材料、新工艺 | P2 |
| **智能检索** | 自然语言问答 | 支持以自然语言提问（如"市政道路沥青面层厚度要求"），AI自动定位相关规范条文并给出解答 | P0 |
| | 规范关联推荐 | 查询某条规范时，自动推荐其引用的其他相关规范 | P1 |
| | 过期规范预警 | 在设计文件中检测引用的规范是否已过期或废止，生成预警清单 | P0 |

#### 3.1.3 数据结构示例

**规范标准条目**

| 字段 | 说明 |
|-----|------|
| 标准编号 | 如 GB 50220-95 |
| 标准名称 | 如《城市道路交通规划设计规范》 |
| 标准级别 | 国标/行标/地标/企标 |
| 发布日期 / 实施日期 | 版本时间信息 |
| 状态 | 现行 / 废止 / 即将实施 |
| 替代关系 | 被哪个新标准替代 |
| 适用专业 | 道路/桥梁/排水/规划/照明等 |
| 适用地区 | 全国 / 具体省市 |
| 全文内容 | 解析后的结构化文本 |
| 条文级索引 | 支持精确到条文级别的检索 |

### 3.2 校审库

> **定位**：建立校审要点清单与常见错误案例库，明确质量红线与标准，实现"经验→规则→自动化检查"的转化。

#### 3.2.1 功能架构

```
校审库
├── 校审要点清单
│   ├── 通用校审要点（全院适用）
│   ├── 专业校审要点（按道路/桥梁/排水/规划等分类）
│   ├── 项目类型校审要点（按市政道路/桥梁/排水/照明等细分）
│   ├── 业主特殊要求清单
│   └── 要点版本管理
│
├── 常见错误案例库
│   ├── 设计错误案例（含问题描述、错误原因、正确做法）
│   ├── 审查意见案例（外审/内审常见意见）
│   ├── 质量事故案例（含整改措施与经验教训）
│   └── 案例标签体系（按专业/错误类型/严重等级分类）
│
├── 质量红线管理
│   ├── 强条违反清单（违反强制性条文的零容忍项）
│   ├── 安全红线（涉及工程安全的底线要求）
│   ├── 合规红线（审批/验收必备条件）
│   └── 红线触发预警机制
│
└── 校审知识图谱
    ├── 规范—要点—案例 关联关系
    ├── 错误根因分析
    └── 改进建议库
```

#### 3.2.2 核心功能清单

| 功能模块 | 功能点 | 功能说明 | 优先级 |
|---------|-------|---------|-------|
| **要点管理** | 校审要点编制 | 支持按专业、项目类型编制结构化校审要点清单，每个要点包含检查内容、判定标准、相关规范依据 | P0 |
| | 要点模板管理 | 预设各专业/项目类型的校审要点模板，项目启动时可一键引用 | P1 |
| | 要点动态更新 | 支持基于外审意见和新规范对校审要点进行持续优化 | P1 |
| **案例管理** | 错误案例录入 | 支持图文并茂地录入错误案例，包含错误截图、问题描述、原因分析、正确做法 | P0 |
| | 案例智能检索 | 支持按专业、关键词、错误类型等多维度检索历史案例 | P1 |
| | 案例推送 | 根据当前设计阶段和专业自动推送相关易错案例给设计人员 | P2 |
| **红线管理** | 强条检测 | 自动识别设计文件中是否违反国家/行业强制性条文 | P0 |
| | 红线预警 | 当设计内容触碰质量红线时，系统实时预警并阻止流程推进 | P0 |
| | 红线统计 | 统计全院红线触碰频次与分布，辅助管理层进行质量决策 | P1 |

## 4. AI工具库板块

AI工具库是面向一线设计人员的智能化工具集，涵盖投标、图审、文审、制图、方案设计五大核心场景。

### 4.1 智能标书中心

> **定位**：投标文件自动编制系统，解决投标文件编制耗时长、质量参差不齐、经验难以复用的核心痛点。

#### 4.1.1 痛点分析

| 痛点 | 具体表现 | 影响 |
|-----|---------|------|
| 编制耗时长 | 一份完整标书通常需3-5人集中编制2-3周，反复修改多轮 | 错失投标窗口期 |
| 质量参差不齐 | 依赖编制人员个人经验，不同人编制的标书水平差异大 | 中标率波动大 |

### 4.5 智能方案设计引擎

> **定位**：专业方案AI辅助设计系统，建立"专业设计数据库 + 规则引擎 + LLM方案生成"三层架构，各专业模块统一接入。

#### 4.5.1 痛点分析

| 痛点 | 具体表现 | 影响 |
|-----|---------|------|
| 方案设计依赖个人经验 | 优秀方案设计高度依赖总工/专业负责人的个人经验 | 人才瓶颈制约产能 |
| 缺乏标准化生成体系 | 方案编制无统一模板，不同人编制水平差异大 | 方案质量不稳定 |
| 计算过程重复耗时 | 结构计算、水文计算等重复性计算占大量时间 | 效率低下 |
| 历史案例难以复用 | 历史方案散落在各分院，无法被新项目有效借鉴 | 经验浪费 |

#### 4.5.2 三层架构设计

```
┌──────────────────────────────────────────────────────┐
│                   第三层：LLM方案生成                   │
│    基于RAG检索相关知识，LLM生成/优化设计方案文本        │
├──────────────────────────────────────────────────────┤
│                   第二层：规则引擎                      │
│    专业设计规则校验、参数约束、合规性检查               │
├──────────────────────────────────────────────────────┤
│                   第一层：专业设计数据库                 │
│    各专业设计参数库、案例库、计算模型库                  │
└──────────────────────────────────────────────────────┘
```

#### 4.5.3 专业模块

##### 模块一：桥梁维修加固方案设计（一院）

| 功能点 | 说明 |
|-------|------|
| 桥梁病害智能识别 | 基于检测报告，AI自动提取桥梁病害信息并分类 |
| 加固方案推荐 | 根据病害类型、桥梁结构形式、荷载等级等参数，推荐加固方案 |
| 计算书自动生成 | 自动生成加固计算书，包含计算过程、计算简图、结果汇总 |
| 方案文本生成 | 生成完整的维修加固方案文本，含工程概况、病害分析、方案比选、工程量等 |
| 造价估算 | 基于历史项目造价数据，自动估算加固方案造价 |

##### 模块二：道路照明设计AI智能体（六院）

| 功能点 | 说明 |
|-------|------|
| 照度计算 | 根据道路等级、路面类型、灯具参数自动计算照度值 |
| 灯具选型 | 基于道路参数和照明标准，自动推荐合适灯具型号及布置间距 |
| 照明方案生成 | 生成道路照明设计方案，含灯具布置图、照度计算书、设备材料表 |
| 节能分析 | 对比不同照明方案的能耗指标，推荐节能优化方案 |
| 标准合规检查 | 检查照明设计是否满足CJJ 45等相关标准要求 |

##### 模块三：污水厂工艺方案AI智能化设计（四院）

| 功能点 | 说明 |
|-------|------|
| 工艺流程推荐 | 根据进出水水质、处理规模等参数，推荐最优污水处理工艺流程 |
| 构筑物选型计算 | 自动进行各处理构筑物（曝气池、沉淀池等）的参数计算与选型 |
| 设备选型 | 根据计算结果自动推荐设备型号（水泵、鼓风机、刮泥机等） |
| 方案文本生成 | 生成完整工艺方案说明书，含工艺流程图、构筑物参数表、设备清单 |
| 经济技术分析 | 自动计算建设投资、运行成本，生成经济技术分析报告 |

##### 模块四：埋地管道抗震验算（六院）

| 功能点 | 说明 |
|-------|------|
| 管道参数输入 | 支持输入管道类型、管径、壁厚、埋深、场地类别等参数 |
| 抗震计算 | 按GB 50032等标准进行管道抗震验算（轴向变形、接口变形等） |
| 计算报告生成 | 自动生成结构化的计算报告，含计算简图、计算过程、结论 |
| 不满足项预警 | 当验算结果不满足要求时，自动预警并给出处理建议 |

##### 模块五：AI造价指标分析助手（造价院）

| 功能点 | 说明 |
|-------|------|
| 造价数据积累 | 沉淀各项目造价数据，建立市政工程造价指标数据库 |
| 造价指标分析 | 按项目类型、地区、规模等维度分析造价指标分布 |
| 方案造价估算 | 在方案阶段快速估算不同方案的投资规模 |
| 同类项目对标 | 将当前项目造价指标与同类项目进行对标分析，识别异常偏差 |
| 造价趋势分析 | 分析各类型工程造价的历史变化趋势，辅助投资决策 |

#### 4.5.4 核心功能清单

| 功能模块 | 功能点 | 功能说明 | 优先级 |
|---------|-------|---------|-------|

## 7. 实施路径与里程碑

### 7.1 实施阶段划分

```
Phase 1 · 基础建设期（第1-3个月）
├── 搭建私有化部署环境
├── 建设LLM推理引擎与RAG系统
├── 启动技术库数据迁移（规范标准入库）
└── 启动校审库要点清单编制

Phase 2 · 知识库完善期（第3-6个月）
├── 完成技术库核心数据入库
├── 完成校审库核心要点与案例录入
├── 启动项目库历史项目归档
└── 知识库智能检索上线

Phase 3 · 工具库一期（第4-9个月）
├── 智能标书中心上线（含模板管理+智能生成+质量检查）
├── 智能图审工场上线（含PDF处理+OCR+规范比对）
├── 智能文审中心上线（含合同审查+文本校审）
└── 用户培训与试运行

Phase 4 · 工具库二期（第7-12个月）
├── CAD智能设计助手上线（统一框架+首批专业规则模块）
├── 智能方案设计引擎上线（首批专业模块接入）
│   ├── 桥梁维修加固方案设计（一院）
│   ├── 道路照明设计智能体（六院）
│   ├── 污水厂工艺方案设计（四院）
│   ├── 埋地管道抗震验算（六院）
│   └── AI造价指标分析助手（造价院）
└── 全平台试运行与优化

Phase 5 · 持续优化期（第12个月以后）
├── 各专业模块持续迭代
├── 知识库持续更新维护
├── 用户反馈驱动功能优化
└── 新专业模块扩展接入
```

| 里程碑 | 时间节点 | 交付内容 |
